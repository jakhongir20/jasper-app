Текущий подход vs SVG из Figma

Сейчас: Части рисуются программно через <path d="...">, <rect>, <line> прямо в коде. Это хорошо для простых фигур, но
ограничено.

Твой вопрос про Figma SVG → JSON: Да, это правильный подход для сложных дизайнов!

Два варианта интеграции SVG с API

Вариант 1: SVG как строка в базе данных

API Response:
{
"id": 1,
"name": "Классическая рамка",
"svg_content": "<path d='M0 0 L100 0...' fill='currentColor'/>"
}

Рендер:
<g dangerouslySetInnerHTML={{ __html: svgContent }} />

Вариант 2: SVG как JSON (более гибко)

{
"id": 1,
"name": "Классическая рамка",
"layers": [
{ "type": "path", "d": "M0 0 L100 0...", "fill": "{{frameColor}}" },
{ "type": "rect", "x": 10, "y": 10, "width": 80, "fill": "#000" }
]
}

Преимущества JSON:

- Можно менять цвета через переменные {{frameColor}}
- Можно масштабировать отдельные слои
- Проще валидация

Рекомендуемая архитектура

┌─────────────────────────────────────────────────────────┐
│ Figma │
│ Дизайнер рисует части двери (рамка, полотно, корона)  │
└─────────────────────┬───────────────────────────────────┘
│ Export SVG
▼
┌─────────────────────────────────────────────────────────┐
│ Конвертер SVG → JSON │
│ Скрипт парсит SVG и создаёт JSON с layers │
└─────────────────────┬───────────────────────────────────┘
│
▼
┌─────────────────────────────────────────────────────────┐
│ API / База данных │
│ products таблица с полем svg_data (JSONB)             │
│ { │
│    "viewBox": "0 0 100 200", │
│    "layers": [...]                                      │
│ } │
└─────────────────────┬───────────────────────────────────┘
│ GET /products?category=5
▼
┌─────────────────────────────────────────────────────────┐
│ React Frontend │
│  <SvgRenderer data={product.svg_data} colors={config}/>│
└─────────────────────────────────────────────────────────┘
